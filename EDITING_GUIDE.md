# Как редактировать без навыков разработки

## Самый простой способ: прямо на GitHub

1. Откройте `https://github.com/Begemot-m/astro-mini-app`.
2. Найдите нужный файл.
3. Нажмите кнопку с карандашом.
4. Измените текст.
5. Нажмите `Commit changes`.
6. Через минуту GitHub Pages обновится автоматически.

## Где менять

- Тексты Mini App: `index.html`
- Поведение кнопок и демо-данные: `app.js`
- Основной дизайн: `styles.css`
- Дополнительные экраны и карта: `enhancements.css`
- Мини-сайт: `site/index.html` и `site/site.css`
- Страница оплаты: `checkout.html`, `checkout.css`, `checkout.js`
- Юридические документы: папка `legal/`
- Claude-промпт: `CLAUDE_PROMPT.md` и `supabase/functions/interpret/index.ts`
- Платежи: `supabase/functions/create-payment/` и `yookassa-webhook/`
- База данных: `supabase/migrations/`

## Что можно менять свободно

- тексты;
- цвета;
- порядок блоков;
- тариф и УТП после согласования документов;
- демо-описания.

## Что нельзя менять без проверки

- старые файлы миграций;
- webhook и проверку платежей;
- RLS-политики;
- секреты и `.env`;
- поля таблиц с подписками и платежами.

## Безопасное правило

Никогда не редактируйте применённую production-миграцию. Для изменения базы создавайте новый файл с следующим номером.
