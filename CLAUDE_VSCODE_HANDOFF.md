# Handoff для продолжения в VS Code / Claude

Проект: `астро.` — прототип Telegram Mini App и мини-сайт для оплаты/документов.

## Что важно

- Основной frontend: `index.html`, `styles.css`, `enhancements.css`, `app.js`, `config.js`.
- Лендинг и документы: `site/`, `legal/`, `checkout.html`.
- Backend-заготовки: `supabase/functions/`, `supabase/migrations/`.
- Платежи спроектированы под ЮKassa через backend, секреты нельзя класть во frontend.
- AI должен вызываться только через backend. Для теста можно подключать GigaChat, для production можно переключить на Claude или другой провайдер.

## Последнее состояние

- Бренд приводится к виду `астро.` с фиолетовой точкой.
- Первый onboarding-экран переписан под приветствие и объяснение продукта.
- Второй onboarding-экран получил roadmap: данные рождения -> расчёт -> AI-интерпретация -> прогноз.
- Иконки Lucide переведены с `unpkg` на `jsdelivr` с fallback, потому что `unpkg` иногда не грузился.
- Добавлен CSS-fallback: если Lucide не загрузится, вместо пустых мест будут фиолетовые точки.

## Что доделать следующим шагом

1. Довести единый стиль `астро.` во всех текстах сайта, документов и paywall.
2. Подключить backend `interpret` к выбранному AI-провайдеру.
3. Настроить Supabase Auth/DB и хранение подписок.
4. Подключить ЮKassa только через backend-функцию `create-payment`.
5. После заполнения юридических реквизитов проверить оферту/политику у юриста.

## Запуск локально

Можно открыть `index.html` напрямую в браузере. Для более честной проверки лучше запустить локальный сервер из папки проекта:

```bash
python -m http.server 4177
```

и открыть:

```text
http://127.0.0.1:4177/
```
