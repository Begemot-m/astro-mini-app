# Пошаговый запуск Астро

Этот план рассчитан на владельца продукта без опыта разработки. Не переходите к реальным платежам, пока шаг не отмечен как завершённый.

## Что уже открыто и редактируемо

- Репозиторий: `https://github.com/Begemot-m/astro-mini-app`
- Mini App: `https://begemot-m.github.io/astro-mini-app/`
- Мини-сайт: `https://begemot-m.github.io/astro-mini-app/site/`
- Checkout: `https://begemot-m.github.io/astro-mini-app/checkout.html`

Код публичный. Тексты и HTML можно редактировать прямо на GitHub через кнопку карандаша. После коммита GitHub Pages обновится автоматически.

## Шаг 1. Оформить бизнес и документы

1. Определить продавца: ИП или ООО.
2. Заполнить реквизиты во всех файлах папки `legal/`.
3. Указать email поддержки и адрес для обращений.
4. Передать документы юристу на проверку.
5. Проверить необходимость уведомления Роскомнадзора как оператора персональных данных.

Готово, когда на сайте нет текста `[ЗАПОЛНИТЬ]`.

## Шаг 2. Подключить Supabase

1. Создать production-проект Supabase.
2. Создать отдельный staging-проект для проверки обновлений.
3. Применить миграции из `supabase/migrations/` сначала на staging, затем на production.
4. Включить ежедневные backups и Point-in-Time Recovery, если доступно на выбранном тарифе.
5. Никогда не хранить данные пользователей или подписки во frontend/GitHub Pages.

Почему подписки не слетят: frontend является заменяемой статикой, а неизменяемая история платежей, периодов подписки и согласий хранится в Postgres.

## Шаг 3. Подключить расчёт карты

1. Развернуть отдельный Python-сервис со Swiss Ephemeris на Railway/Fly.io.
2. Указать его URL в секрете `CHART_CALC_URL`.
3. Проверить минимум 10 карт по независимому референсу.
4. Только после проверки убрать маркировку demo из интерфейса.

## Шаг 4. Подключить Claude API

1. Создать API-ключ Anthropic.
2. Сохранить его как Supabase Secret `ANTHROPIC_API_KEY`.
3. Задеплоить функцию `interpret`.
4. Проверить тексты на запрещённые диагнозы, обещания и категоричные прогнозы.
5. Настроить месячные лимиты и кэширование.

## Шаг 5. Подключить ЮKassa

1. Зарегистрировать магазин ЮKassa на ваше ИП/ООО.
2. Передать ЮKassa URL мини-сайта и заполненные юридические страницы.
3. Получить `shopId` и секретный ключ.
4. Сохранить их только в Supabase Secrets.
5. Настроить webhook на функцию `yookassa-webhook`.
6. Провести тестовый платёж, отмену автопродления, повторное списание и возврат.
7. Получить подтверждение ЮKassa по автоплатежам.

## Шаг 6. Подключить Telegram

1. Создать бота через BotFather.
2. Установить Menu Button на URL Mini App.
3. Проверять подпись Telegram `initData` только на backend.
4. Не доверять Telegram ID, переданному обычным JSON из frontend.
5. Настроить webhook бота и утренние уведомления.

## Шаг 7. Первый ограниченный запуск

1. Включить регистрацию для 20–50 тестовых пользователей.
2. Платежи сначала оставить в тестовом режиме.
3. Собрать обратную связь по точности карты и ощущению «это про меня».
4. Проверить расходы Claude и время генерации.
5. После проверки включить реальные платежи feature-флагом.

## Безопасное обновление

1. Изменения сначала проверяются на staging.
2. Перед миграцией создаётся backup.
3. Миграции только добавляют структуру; существующие поля не удаляются и не переименовываются в том же релизе.
4. Платёжные webhook-события сохраняются до обработки и имеют уникальный ID.
5. Каждый успешный платёж создаёт неизменяемый период подписки.
6. Новую функцию можно отключить feature-флагом без удаления данных.

## Что нельзя публиковать

- `.env` с ключами;
- `SUPABASE_SERVICE_ROLE_KEY`;
- ключ Claude;
- секрет ЮKassa;
- дампы пользовательской базы;
- логи с персональными данными.
