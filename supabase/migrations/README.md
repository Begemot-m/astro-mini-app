# Правила миграций

- Миграции применяются по порядку и никогда не редактируются после production-запуска.
- Обычный релиз не удаляет и не переименовывает таблицы, колонки или enum-значения.
- Сначала добавляется новая структура, затем код начинает её использовать.
- История платежей, периодов подписки и согласий не удаляется каскадно.
- Перед каждой production-миграцией создаётся backup и выполняется проверка на staging.
- Service role хранится только в Supabase Secrets.
