# Безопасная архитектура Астро

## Расчёт и Claude AI

1. Клиент передаёт данные рождения на backend.
2. Backend геокодирует место и определяет исторический часовой пояс.
3. Swiss Ephemeris рассчитывает положения планет, дома, аспекты и ретроградность.
4. Backend формирует обезличенный JSON карты без Telegram ID, username и точного адреса.
5. Claude API получает JSON и генерирует только интерпретацию по строгому системному промпту.
6. Ответ проверяется фильтрами безопасности и кэшируется.

Шаблон контракта для Claude находится в `CLAUDE_PROMPT.md`. Статический текст прототипа должен заменяться структурированным ответом функции `interpret`.

Claude не должен рассчитывать положения планет, ставить диагнозы, обещать события или результат, а также давать медицинские, юридические или инвестиционные рекомендации.

Рекомендуемый тон: «может проявляться», «обратите внимание», «проверьте по своему опыту», «возможный ориентир».

## Оплата через ЮKassa

В выбранной продуктовой схеме Mini App переводит пользователя на самостоятельную внешнюю веб-страницу оплаты ЮKassa. Это следует запускать только после отдельной проверки соответствия правилам Telegram и консультации с юристом.

Сценарий:

- отдельный HTTPS-домен и публичная оферта;
- создание платежа только через backend Edge Function;
- секретный ключ ЮKassa никогда не попадает во frontend;
- уникальный ключ идемпотентности на каждый платёж;
- redirect на защищённую страницу ЮKassa;
- доступ открывается только после webhook и повторной проверки платежа через API ЮKassa;
- явное согласие на автоплатёж;
- отображение суммы и периодичности списаний;
- самостоятельное отключение автопродления;
- чеки, возвраты и синхронизация подписки с backend.
- регулярное продление отдельной cron-функцией;
- самостоятельное отключение автопродления без потери уже оплаченного периода.

Важно: Telegram требует Stars для цифровых услуг, продаваемых внутри Telegram. Внешняя ЮKassa-схема несёт платформенный риск и должна быть осознанным бизнес-решением.

## Юридическая рамка

Перед запуском нужны проверенные юристом пользовательское соглашение, публичная оферта, политика обработки персональных данных, согласие на обработку данных рождения, правила возвратов и возрастные ограничения.

Интерфейс должен постоянно описывать сервис как развлекательно-рефлексивный и не заменяющий профессиональную помощь.
