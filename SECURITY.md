# Security policy

## Секреты

Секреты хранятся только в Supabase Secrets. Они никогда не коммитятся в GitHub и не передаются frontend.

## Персональные данные

- Frontend не получает service role.
- Claude получает обезличенный JSON.
- RLS включён для пользовательских таблиц.
- Платёжные события изменяются только service role функциями.

## Платежи

- Событие webhook сначала записывается в `payment_events`.
- Повторное событие безопасно игнорируется по уникальному ID.
- Платёж повторно проверяется через API ЮKassa.
- Доступ определяется по неизменяемым `subscription_periods`.
- Таблица `subscriptions` является удобным текущим состоянием, но не единственным источником истины.

## Обновления

- Frontend не хранит состояние подписок.
- Миграции только аддитивные.
- Production-миграции сначала проверяются на staging.
- Перед миграцией создаётся backup.
